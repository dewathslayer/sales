Sales Data Analysis
Project Overview

This project contains data analysis focused on sales and revenue trends. It includes SQL notebooks, data exploration, and visualizations that help understand how sales vary across categories, dates, and regions. The goal is to analyze net revenue, identify patterns, and demonstrate SQL-based analytical workflows.

Contents

SQL_Notebook.ipynb – Main SQL queries and analysis.

Tablesused.ipynb – Description of tables used and data processing steps.

datas_using_aggregatefunctions.ipynb – Analysis using aggregate functions such as SUM, AVG, COUNT, MAX, and MIN.

regionsSQL_Notebook.ipynb – Region-based revenue and sales insights.

total_net_revenue_basedondatesSQL_Notebook.ipynb – Trend analysis of net revenue based on dates.

Key Features

Net revenue comparison between years (e.g., 2022 vs 2023).

Category-wise analysis of sales performance.

Date-based revenue insights to identify rising or declining trends.

SQL-driven approach for extracting, transforming, and analyzing data.

Clean and reproducible notebooks for learning or extending analysis.

Getting Started

Clone the repository:
git clone https://github.com/dewathslayer/sales.git

cd sales

Open the Jupyter notebooks in JupyterLab, VS Code, or Google Colab.

Ensure you have your SQL environment or database connected.

Run the cells to view queries, results, and visualizations.

Requirements

Python 3.x

Jupyter Notebook or JupyterLab

SQL database connection (depending on your setup)

Python libraries such as pandas and matplotlib (if used inside notebooks)
